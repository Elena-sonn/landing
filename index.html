How it works?

What any application actually is? It consist of just 3 things: user interface, data and rules of how user actions changes that data. In Skem.io you describe those 3 things in 3 steps.

Design

Build the interface of your application. We didn't invent a wheel here: you just add elements into the view, align and style them as in any site builder, mock up application or graphic editor.

Data

Connect your data to interface. Just drag'n'drop pieces of data to properties of interface components, once data is connected interface will be updated automatically every time data changes.

Action

Define how user actions changes the data by selecting a trigger (such as click, key press etc.) and describing "action" - a rule of how data transfers from current state to next one.
